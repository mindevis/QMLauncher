# История изменений

Все значимые изменения в QMLauncher документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект следует [Semantic Versioning](https://semver.org/lang/ru/).

## [1.0.0] - 2025-01-11

### Добавлено
- **Полная миграция с Electron на Wails**
  - Go бэкенд с интеграцией Wails v2
  - React + TypeScript фронтенд с компонентами shadcn/ui
  - Нативная производительность и меньший размер приложения

- **Установка клиента Minecraft**
  - Автоматическая загрузка и установка JAR файла клиента Minecraft
  - Управление библиотеками с параллельной загрузкой
  - Загрузка ресурсов с проверкой SHA1
  - Извлечение нативных библиотек (LWJGL) для Windows, Linux и macOS
  - Обработка платформо-специфичных нативных библиотек
  - Отслеживание прогресса установки

- **Управление Java Runtime**
  - Автоматическая установка Java (OpenJDK)
  - Выбор вендора и версии для каждого сервера
  - Логика повторных попыток с экспоненциальной задержкой для загрузок
  - Обработка временных файлов для предотвращения поврежденных загрузок
  - Валидация и кэширование пути к Java

- **Запуск клиента Minecraft**
  - Полное построение JVM аргументов
  - Обработка игровых аргументов с подстановкой переменных
  - Автоматическое подключение к серверу через аргументы `--server` и `--port`
  - Поддержка `--quickPlayMultiplayer` для современных версий Minecraft
  - Нормализация путей (разворачивание тильды)
  - Управление рабочей директорией

- **Управление серверами**
  - Отображение списка серверов с функцией поиска
  - Отслеживание статуса установки сервера
  - Установка клиента для каждого сервера
  - Настройки для каждого сервера (путь Minecraft, путь Java, JVM аргументы, память)
  - Удаление сервера с правильной очисткой
  - Отображение иконок загрузчиков (Forge, Fabric, Quilt, NeoForge, Vanilla)

- **Аутентификация и управление сессией**
  - Форма входа с логином/паролем
  - Аутентификация на основе токенов
  - Сохранение сессии в localStorage
  - "Мягкий" офлайн режим - сохраняет токен, но показывает предупреждение при недоступности сервера
  - Валидация токена в фоновом режиме
  - Автоматическое обновление токена при запуске приложения

- **Пользовательский интерфейс**
  - Современный UI на React, TypeScript и Tailwind CSS
  - Интеграция библиотеки компонентов shadcn/ui
  - Поддержка интернационализации (i18n) (русский и английский)
  - Уведомления (toast) через sonner
  - Прогресс-бары для установки
  - Модальные окна для подтверждений
  - Адаптивный дизайн

- **Функции для Windows**
  - Скрытие консольного окна с использованием `javaw.exe` и флага `CREATE_NO_WINDOW`
  - Подавление вывода процесса на Windows
  - Обработка путей специфичная для Windows

- **Управление процессами**
  - Отслеживание процесса Minecraft
  - Автоматическое обнаружение завершения процесса
  - Управление состоянием кнопки Play на основе запущенных процессов
  - Периодическая проверка статуса процесса

- **Обработка ошибок и логирование**
  - Подробное логирование по всему приложению
  - Детальные сообщения об ошибках для пользователей
  - Обработка сетевых ошибок с повторными попытками
  - Обработка ошибок файловой системы
  - Восстановление после ошибок установки

- **Управление конфигурацией**
  - Кэширование конфигурации сервера
  - Поддержка встроенных серверов
  - Сохранение настроек лаунчера
  - Управление UUID сервера для правильного удаления

### Изменено
- **Архитектура**: Миграция с Electron (Node.js) на Wails (Go + WebView)
- **IPC коммуникация**: Замена Electron IPC на методы контекста Wails
- **Файловые операции**: Миграция с Node.js fs на Go os пакет
- **Управление процессами**: Миграция с Node.js child_process на Go exec пакет
- **Система сборки**: Замена Electron Builder на систему сборки Wails

### Исправлено
- **Извлечение нативных библиотек**: Исправлена проблема загрузки DLL LWJGL на Windows
- **Обработка путей**: Исправлены проблемы с разворачиванием тильды и нормализацией путей
- **Загрузка ресурсов**: Исправлена ошибка "Can't open the resource index file"
- **Сохранение сессии**: Исправлена проблема с сохранением токена при перезапуске приложения
- **Удаление**: Исправлено определение UUID сервера для правильной очистки
- **Консольное окно**: Исправлено мигающее консольное окно на Windows при запуске Minecraft
- **Состояние кнопки Play**: Исправлена проблема с кнопкой, остающейся отключенной после закрытия клиента
- **Подключение к серверу**: Исправлено автоматическое подключение к серверу при запуске
- **Загрузка Java**: Исправлены ошибки 503 и загрузки файлов нулевого размера с логикой повторных попыток

### Технические детали

#### Бэкенд (Go)
- `app.go`: Основная структура приложения и контекст Wails
- `minecraft.go`: Установка, запуск и управление клиентом Minecraft
- `java.go`: Установка и управление Java runtime
- `config.go`: Управление конфигурацией и кэширование данных сервера
- `mods.go`: Управление модами (подготовлено для будущей реализации)

#### Фронтенд (React + TypeScript)
- `App.tsx`: Главный компонент приложения с аутентификацией и маршрутизацией
- `ServersTab.tsx`: Список серверов, установка и управление запуском
- `ServerSettingsDialog.tsx`: Управление настройками для каждого сервера
- `bridge.ts`: TypeScript мост для методов Go бэкенда
- `locales/`: Файлы интернационализации (ru.json, en.json)

#### Ключевые особенности реализации
- **Извлечение нативных библиотек**: Извлекает файлы `.dll`, `.so`, `.dylib` из JAR архивов в зависимости от платформы
- **Автоматическое подключение к серверу**: Использует как `--server`/`--port`, так и `--quickPlayMultiplayer` для совместимости
- **Скрытие процесса**: Использует `syscall.SysProcAttr` с флагом `CREATE_NO_WINDOW` на Windows
- **Сохранение сессии**: Сохраняет токен авторизации в localStorage и валидирует в фоновом режиме
- **Логика повторных попыток**: Реализует экспоненциальную задержку для сетевых запросов
