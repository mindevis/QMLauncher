# История изменений

Все значимые изменения в QMLauncher документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект следует [Semantic Versioning](https://semver.org/lang/ru/).

## [1.0.3-rc1] - 2025-12-16

### Добавлено
- **Улучшенное отображение версий**: Улучшено отображение версий лаунчера в QMAdmin с информацией о коммитах и датах сборки
- **Автоматическое определение версии**: QMServer теперь автоматически получает версию из GitHub релизов при сборке лаунчера
- **Интеграция сообщений коммитов**: Сообщения коммитов теперь отображаются вместе с информацией о версии в списках сборок

### Изменено
- **Формат версии**: Обновлен формат отображения версии на `QMLauncher v1.0.0 (2e83ccd) - 16.12.2024` для лучшей читаемости
- **Статус сборки**: Расширен статус сборки для включения сообщения коммита, информации о сервере и улучшенных метаданных

### Технические детали
- Версия автоматически получается из GitHub Releases API, если не указана явно
- Сообщения коммитов обрезаются до 40 символов для отображения
- Даты сборки форматируются в локализованном формате (ru-RU)

## [1.0.3] - 2025-12-16

### Добавлено
- **Обфускация кода**: Добавлена обфускация кода через garble для всех платформ для защиты от реверс-инжиниринга
  - Включается через флаг `-obfuscated` в команде wails build
  - Применяется к сборкам для Windows, Linux и macOS
  - Требует инструмент garble (автоматически устанавливается в окружении сборки QMServer)
- **Встроенная зашифрованная конфигурация**: Конфигурация теперь встраивается непосредственно в исполняемый файл (`.exe`)
  - Конфигурация шифруется с помощью AES-256-GCM перед встраиванием
  - Автоматически расшифровывается и копируется в пользовательскую директорию при первом запуске
  - Доступна только когда лаунчер собирается через QMServer (Режим 3)
- **Три режима работы**: Лаунчер теперь поддерживает три различных режима работы
  - **Режим 1 (Автономный)**: Работает без подключения к QMServer, пользователь выбирает режим при первом запуске
  - **Режим 2 (Ручной)**: Пользователь вручную вводит адрес QMServer при первом запуске
  - **Режим 3 (Встроенный)**: Предварительно настроен при сборке через QMServer, диалог выбора режима не показывается

### Изменено
- **Процесс сборки**: Удалена поддержка NSIS установщика, теперь создаются исполняемые файлы напрямую
  - Windows: Создает файл `.exe` вместо установщика `.msi`
  - Все платформы используют флаг `-obfuscated` для защиты кода
- **Встраивание конфигурации**: Конфигурация теперь встраивается в бинарник через директиву Go `embed` вместо внешних файлов
  - Использует файл `embedded_config.json`, который компилируется в исполняемый файл
  - Конфигурация шифруется перед встраиванием для безопасности
- **Скрипты сборки**: Обновлен `build-windows.sh` для использования флага `-obfuscated` вместо `-nsis`
- **Документация**: Обновлены `BUILD.md` и `README.md` для отражения обфускации и удаления упоминаний NSIS

### Безопасность
- **Зашифрованная встроенная конфигурация**: Конфигурация, встроенная в лаунчер, собранный через QMServer, шифруется с помощью AES-256-GCM
  - Использует фиксированный ключ шифрования, известный как QMServer, так и QMLauncher
  - Защищает чувствительные данные конфигурации сервера от случайного просмотра

### Технические детали
- Встроенная конфигурация хранится в `embedded_config.json` и компилируется в бинарник с помощью `//go:embed`
- Расшифровка конфигурации происходит автоматически при первом запуске, если встроенная конфигурация существует
- Обфускация кода применяется ко всем сборкам через QMServer для повышения безопасности
- Определение режима: Лаунчер автоматически определяет встроенную конфигурацию и пропускает диалог выбора режима

## [1.0.2] - 2025-12-15

### Добавлено
- **Поддержка установочных пакетов**: Добавлена поддержка создания установочных пакетов для разных операционных систем
  - Windows: NSIS установщик (файл `.msi`) через флаг `-nsis`
  - Linux: AppImage пакет (файл `.AppImage`)
  - macOS: Образ диска (`.dmg`) или пакет (`.pkg`)
- **Поддержка встроенной конфигурации**: Автоматическое копирование встроенного `config.json` при сборке лаунчера через QMServer
  - Конфигурация автоматически копируется из директории сборки при первом запуске
  - Поддержка различных мест установки (app bundle, директория исполняемого файла)
  - Возврат к настройкам по умолчанию, если встроенная конфигурация недоступна
- **Автоматическое определение UUID сервера**: Автоматическое получение UUID сервера от QMServer при подключении в режиме сервера
  - Получает список серверов из настроенного API endpoint
  - Извлекает UUID из первого доступного сервера
  - Сохраняет UUID в локальные настройки для правильной идентификации сервера

### Изменено
- **Скрипты сборки**: Обновлен `build-windows.sh` для использования флага `-nsis` при создании установщика
- **Документация сборки**: Расширен `BUILD.md` с инструкциями по сборке установочных пакетов для всех платформ
- **Управление конфигурацией**: Улучшен `config.go` для обработки встроенных файлов конфигурации из сборок QMServer
- **Конфигурация Wails**: Обновлен `wails.json` для использования `"nsisType": "nsis"` при создании Windows установщика

### Технические детали
- Создание NSIS установщика теперь является стандартным для Windows сборок
- Встроенная конфигурация ищется в нескольких местах (директория исполняемого файла, app bundle, директория сборки)
- UUID сервера автоматически получается при подключении лаунчера к QMServer в режиме сервера
- Копирование конфигурации происходит прозрачно при первом запуске, если встроенная конфигурация существует

### Технические детали

#### Бэкенд (Go)
- `app.go`: Основная структура приложения и контекст Wails
- `minecraft.go`: Установка, запуск и управление клиентом Minecraft
- `java.go`: Установка и управление Java runtime
- `config.go`: Управление конфигурацией и кэширование данных сервера
- `mods.go`: Управление модами (подготовлено для будущей реализации)

#### Фронтенд (React + TypeScript)
- `App.tsx`: Главный компонент приложения с аутентификацией и маршрутизацией
- `ServersTab.tsx`: Список серверов, установка и управление запуском
- `ServerSettingsDialog.tsx`: Управление настройками для каждого сервера
- `bridge.ts`: TypeScript мост для методов Go бэкенда
- `locales/`: Файлы интернационализации (ru.json, en.json)

#### Ключевые особенности реализации
- **Извлечение нативных библиотек**: Извлекает файлы `.dll`, `.so`, `.dylib` из JAR архивов в зависимости от платформы
- **Автоматическое подключение к серверу**: Использует как `--server`/`--port`, так и `--quickPlayMultiplayer` для совместимости
- **Скрытие процесса**: Использует `syscall.SysProcAttr` с флагом `CREATE_NO_WINDOW` на Windows
- **Сохранение сессии**: Сохраняет токен авторизации в localStorage и валидирует в фоновом режиме
- **Логика повторных попыток**: Реализует экспоненциальную задержку для сетевых запросов
