# Code Review: Оптимизация, Безопасность и Очистка

## 🔒 Проблемы безопасности

### 1. Хардкод accessToken
**Файл:** `frontend/src/components/ServersTab.tsx:777`
```typescript
'--accessToken', 'token',  // ❌ НЕБЕЗОПАСНО: хардкод токена
```

**Проблема:** Используется статический токен `'token'` вместо реального токена пользователя.

**Решение:** 
- Убрать этот аргумент, если он не нужен для офлайн режима
- Или использовать реальный токен из localStorage/API

### 2. Статический ключ шифрования
**Файл:** `frontend/src/utils/secureStorage.ts:7`
```typescript
const ENCRYPTION_KEY = 'qmlauncher_encryption_key' // ❌ Статический ключ
```

**Проблема:** Статический ключ шифрования легко извлечь из кода.

**Решение:**
- Генерировать ключ на основе HWID или других уникальных данных устройства
- Или использовать Go backend для шифрования (более безопасно)

### 3. Токены в localStorage
**Проблема:** Токены хранятся в localStorage, что уязвимо для XSS атак.

**Текущее состояние:** Используется обфускация, но это не полноценное шифрование.

**Рекомендация:** 
- Рассмотреть использование Go backend для безопасного хранения токенов
- Или использовать более стойкое шифрование

## 🧹 Неиспользуемый код

### 1. secureStorage.ts не используется
**Файл:** `frontend/src/utils/secureStorage.ts`

**Проблема:** Файл создан, но не используется. Вместо него используется прямой доступ к localStorage:
```typescript
localStorage.getItem('qmlauncher_auth_token')
localStorage.setItem('qmlauncher_auth_token', token)
```

**Решение:**
- Либо использовать secureStorage везде
- Либо удалить файл, если обфускация не нужна

### 2. logger.ts частично используется
**Файл:** `frontend/src/utils/logger.ts`

**Проблема:** Создан logger, но в коде осталось много `console.log/error/warn`.

**Статистика:** Найдено 105+ вызовов `console.*` в коде.

**Решение:**
- Заменить все `console.log` на `logger.debug` (только для dev)
- Заменить `console.error` на `logger.error`
- Удалить отладочные логи из production кода
